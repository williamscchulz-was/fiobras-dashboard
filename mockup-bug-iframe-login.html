▼ Por que isso acontece

Histórico: o sub-app Manutenção foi originalmente STANDALONE (rodava sozinho em manutencao.fiobras.com.br) com seu próprio sistema de login. Quando foi integrado como iframe no HUB v3.8.0, o bootstrap __hubBootManut() foi criado pra "auto-logar como admin" e pular a tela de login interna.

Existe uma função _autoLoginViaHub() que TENTA pegar o usuário do HUB via window.getCurrentUser(). Mas essa função vive no contexto do parent (HUB), não no iframe. Dentro do iframe, window.getCurrentUser é undefined.

Resultado: _autoLoginViaHub() sempre falha → cai no __hubBootManut() que força admin. QUALQUER user que entrar via HUB, dentro do Manutenção é tratado como admin.

▼ Solução proposta

🔑 Iframe lê a sessão DIRETO do localStorage

Iframes no MESMO domínio (subdomínio) compartilham o mesmo localStorage. Como HUB e Manutenção rodam em hub.fiobras.com.br (HUB) e hub.fiobras.com.br/manutencao/ (iframe), o iframe pode ler localStorage["fiobras-dash-auth"] diretamente.

Substituir _autoLoginViaHub() por uma versão que lê localStorage:

function _autoLoginViaHub() {
  let sess = null;
  try {
    sess = JSON.parse(localStorage.getItem('fiobras-dash-auth'));
  } catch(e) {}
  if (!sess || !sess.user) return false;

  // Pega role do users-config + users-profile (cache do localStorage)
  const cfg = state.usersConfig || {};
  const profs = state.userProfiles || {};
  const c = cfg[sess.user] || {};
  const p = profs[sess.user] || {};
  const role = p.roleOverride || c.role || sess.role || 'producao';

  state.user    = sess.nome || c.nome || p.nomeCompleto || sess.user;
  state.userKey = sess.user;
  state.isAdmin = (role === 'admin' || role === 'gerente');

  // ... resto do init
  return true;
}

E remover o force admin do __hubBootManut(): se _autoLoginViaHub() retornar true, não faz mais nada. Só cai no fallback de admin se não houver sessão (ex: alguém abriu manutencao/ direto sem passar pelo HUB).

▼ Plano de implementação

  • Reescrever _autoLoginViaHub() pra ler localStorage["fiobras-dash-auth"] direto, sem depender de window.getCurrentUser
  • Remover o force-admin do __hubBootManut(). Só faz login local se sessão localStorage não existir.
  • Ler role real do users-profile/<userKey>/roleOverride + users-config/<userKey>/role (cache localStorage cobre o caso de Firebase ainda não carregado)
  • Setar state.isAdmin CORRETAMENTE baseado no role real (admin OU gerente → true; producao → false)
  • Sub-apps CRM e Preço também têm o mesmo problema potencial — auditar e aplicar mesmo fix se necessário
