▼ O que muda tecnicamente

1. Cache imediato (sem flash de login)
O checkAuth() roda antes de qualquer render do .login-screen. Tela de login só aparece se cache ausente/expirado. Adicionar classe .hide no HTML do login antes do JS rodar (default hidden, mostra se necessário) — elimina o flash.

2. Splash personalizado "Bem-vindo de volta, <Nome>"
Em vez de só ir pro splash genérico, mostra "Bem-vindo de volta" + nome + foto/iniciais do user. Confirma visualmente que é a conta certa. Duração: 600-800ms (rápido o suficiente pra não atrasar, lento o suficiente pra registrar).

3. Botão "Não é você? Trocar conta"
Discreto no rodapé do splash de boas-vindas. Click → limpa cache + mostra login. Crucial pra device compartilhado (PC do laboratório, tablet do chão).

4. Renovação automática do TTL no boot
Cada auto-login renova o TTL pra +30 dias. User ativo nunca expira; user inativo por 30d cai no login. Hoje já faz isso indiretamente via setSession() ao mover entre módulos — mas garantir que renove no boot também.

5. Cache estruturado (compatível com o atual)
Mantém localStorage["fiobras-dash-auth"] com schema atual: {user, role, nome, ts, expires, remember}. Sem mudança breaking.

▼ Edge cases tratados

✓ Cache válido
Splash "Bem-vindo de volta" → renova TTL → entra direto no último módulo aberto.
⏱ Cache expirado
Toast amarelo "Sua sessão expirou após 30 dias inativo" + tela de login com user pré-selecionado.
✕ Force logout (admin)
Admin clicou "Forçar logout" → force-logout/<user> no Firebase é detectado → cache limpo → tela de login.
👥 Trocar conta
Click no "Não é você?" no splash → cache limpo → tela de login com dropdown vazio.
🚫 User desativado
Boot detecta profile.active === false → cache limpo + toast "Usuário inativo. Contate o admin." + tela de login.
🔄 Senha trocada
User mudou senha em outro device → cache local fica válido até expirar OU até admin forçar logout. Aceitável (sessão local pré-troca).
