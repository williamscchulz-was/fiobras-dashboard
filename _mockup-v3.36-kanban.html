5 · Implementação

Schema dos cards — manutencao/kanban/{id}

  • Existentes: titulo, desc, equip, setor, tipo, urg, col, data, autor, prevId, resp, log[]
  • NOVOS opcionais:
    • horasEstimadas: number (horas, ex: 2.5). Vira chip "Est. 2h30" no footer quando col ≠ doing/done.
    • foto: já existe (string base64 ou URL) — vira thumbnail top do card. Já é salva ao criar demanda.
    • envolvidos: array de chaves de user. Se não tiver, stack mostra só o resp.
    • progresso: number 0-100 (manual via slider no modal de edição). Default: 0/50/100 conforme col (todo/doing/done).
  • Retrocompat: todos os campos novos são opcionais; cards antigos renderizam sem eles.

CSS

  • Novas classes .kc-* (kanban card) substituindo .kanban-card atual — nome diferente pra não quebrar cards legacy
  • Light/dark via [data-theme="..."] (segue padrão do HUB)
  • Board tem padding 22px + fundo dark independente do tema (#0F1114 coeso com a estética "industrial ativa")

JS · render

  • renderKanban() reescrito — usa helpers: _renderCard(c), _relativeDate(ts), _fmtTimer(ms)
  • Timer ativo recalcula a cada 1s via _tickTimers() (já existe na v3.32.3)
  • Drag/drop mantém HTML5 nativo (desktop) + botões ✈ mobile (coerente com CRM v3.21.13)

Modal · campos novos

  • Modal de criação/edição de demanda ganha: Horas estimadas (number input), Progresso (slider 0-100 só no modo edição)
  • Campo Envolvidos (multi-select de users) — opcional, fallback só resp
