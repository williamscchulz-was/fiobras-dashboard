5 · "Adicionar tarefa nova" (custom)

Se a tarefa desejada não está na lista padrão da máquina, user digita no campo do rodapé → vira um item novo já marcado na lista (badge verde "+ADIC.").

Bonus: tarefas custom criadas aqui podem (opcional) ser salvas no m.prevs da máquina pra aparecer como padrão nas próximas. Pode virar um checkbox "Salvar essa tarefa como padrão dessa máquina" ao lado. Decidir depois.

7 · Implementação

HTML · modal #modal-preventiva

  • Campo #pv-maquina continua select single (sem mudança)
  • Campo #pv-tarefa-sel (select single) → substituído por .ts-wrap com checkboxes + input "adicionar"
  • Lista popula dinamicamente ao trocar máquina: pega m.prevs + eventuais tarefas custom acumuladas
  • Summary .ts-summary + botão salvar atualizam reativo ao selecionar

JS · fluxo salvar

  • Criar: for (const tarefa of tarefasSel) window._fbPushPrev({...base, tarefa})
  • Editar: campo texto normal → _fbUpdatePrev(key, {...dados})
  • Validação: if (!tarefasSel.length) return alert('Selecione 1+ tarefa')
  • Toast: "N preventivas cadastradas" (plural) ou "Preventiva cadastrada"
