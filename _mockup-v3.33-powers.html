3 · Schema Firebase

  • Path: users-profile/{user}/powers/{poder} = true | false
  • Helper global: window.canUser(userKey, 'deletarPreventiva')
  • Lógica do helper:
    1. Se role = admin OU gerente → sempre true (bypass)
    2. Senão: lê users-profile/{userKey}/powers/{poder}
    3. Se nó não existe → usa POWER_DEFAULTS[poder].producao
  • Seed inicial roda 1x (flag em localStorage fiobras-powers-seed-v1):
    • Pra cada user em USERS hardcoded + dinâmicos: aplica defaults da role
    • Override do Joacir: powers/deletarPreventiva = true
    • Não roda se chave já existe
  • Sub-apps (Manutenção/CRM/Preço) leem do mesmo path via onValue e mantêm em state.powers[user]

4 · Aplicação prática (Ajuste 3b)

  • Tabela de Preventivas (renderPreventiva): botão Deletar só sai se canUser(currentUser, 'deletarPreventiva') retornar true
  • Cards mobile: mesmo gate
  • Função deletarPreventiva: dupla checagem antes de remover (defense in depth)
  • Fluxo: user sem o poder não vê o botão; se chamar via console, função recusa com toast "Sem permissão"
